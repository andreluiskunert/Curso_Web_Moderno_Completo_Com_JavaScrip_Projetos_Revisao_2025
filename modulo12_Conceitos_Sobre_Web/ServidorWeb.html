Resumo: Servidores Web

O que é um Servidor Web?

É um software que recebe requisições HTTP e retorna respostas, como páginas HTML, CSS, imagens e dados dinâmicos.

  • Exemplos: Apache, Nginx, Tomcat, Node.js (Express), Spring Boot.
  • Pode servir conteúdo estático (imagens, CSS, JS) ou dinâmico (gerado por linguagens como PHP, Java, Node.js).

Estratégias de Processamento

  • Apache: abre múltiplas threads, uma para cada requisição.
  • Nginx: trabalha de forma assíncrona, usando poucas threads.
  • Node.js: utiliza um loop de eventos com processamento assíncrono.

DNS (Domain Name System)

Responsável por traduzir um domínio (ex: www.exemplo.com) para um endereço IP do servidor.

  • Configuração feita em serviços como Registro BR ou provedores externos.
  • Propagação de alterações pode levar de 24h a 48h.

Hospedagem e Escalabilidade

  • Um servidor pode atender múltiplos sites via virtual hosts.
  • Sites grandes utilizam balanceamento de carga e clusters.
  • Conteúdo estático pode ser entregue via CDN para maior performance.

Pilha de Servidores

Muitas vezes o servidor web é parte de um conjunto maior:

  • LAMP: Linux + Apache + MySQL + PHP
  • XAMPP/MAMP/WAMP: Pacotes que instalam tudo junto.

Portas de Comunicação

  • HTTP: Porta 80
  • HTTPS: Porta 443
  • Cada aplicação só pode usar uma porta por vez.
