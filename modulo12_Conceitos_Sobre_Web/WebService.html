Resumo sobre Web Services

O que são Web Services?

São serviços disponibilizados na web que permitem a comunicação entre diferentes sistemas e linguagens, geralmente via HTTP. Eles fornecem funções de negócio (como cadastro de clientes) de forma padronizada.

Por que usar?

  • Facilitam a interoperabilidade entre sistemas.
  • Usam formatos textuais (XML, JSON) para troca de dados.
  • Mais simples do que formatos binários proprietários.
  • Base para arquiteturas modernas como SOA e Microserviços.

Principais tipos

  • SOAP (Simple Object Access Protocol) - Baseado em XML, burocrático, usa WSDL para descrever serviços.
  • REST (Representational State Transfer) - Mais simples e leve, usa diretamente os verbos do HTTP (GET, POST, PUT, DELETE).

CRUD no REST

O REST segue as operações básicas do banco de dados:

  • POST → Criar (Create)
  • GET → Ler (Read)
  • PUT → Atualizar (Update)
  • DELETE → Excluir (Delete)

Arquiteturas Relacionadas

  • SOA (Service Oriented Architecture): Serviços centralizados em um barramento (ESB).
  • Microserviços: Aplicações divididas em módulos menores, independentes e escaláveis.

Web Services = Integração + Padrão + Flexibilidade
