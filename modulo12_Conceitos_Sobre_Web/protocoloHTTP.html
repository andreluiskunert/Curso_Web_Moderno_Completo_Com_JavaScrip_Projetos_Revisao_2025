Resumo do Protocolo HTTP

Protocolo de Transferência de Hipertexto

O que é HTTP?

O HTTP (Hypertext Transfer Protocol) é um protocolo da camada de aplicação usado na web para comunicação entre cliente (navegador) e servidor. Originalmente foi criado para transferir hipertextos (documentos HTML com links), mas hoje suporta imagens, vídeos, áudios, jogos e muito mais.

Características principais

  • Camada: Funciona na camada de aplicação da pilha TCP/IP.
  • Sem estado (stateless): Cada requisição é independente. Sessões são geridas pela aplicação.
  • Cliente-Servidor: O cliente faz requisições e o servidor responde.
  • Baseado em requisição/resposta: Cliente envia a request e recebe uma response.

Métodos HTTP

  • GET: Recupera informações (parâmetros vão na URL).
  • POST: Envia dados no corpo da requisição (ex.: formulários).
  • PUT, DELETE, PATCH, OPTIONS, CONNECT, HEAD também existem e são usados em APIs.

Status de Resposta

  • 1xx: Informativo
  • 2xx: Sucesso (200 OK)
  • 3xx: Redirecionamento (301, 302...)
  • 4xx: Erros do cliente (404 Not Found)
  • 5xx: Erros do servidor (500 Internal Server Error)

HTTP x HTTPS

O HTTPS é a versão segura do HTTP. Ele usa criptografia (SSL/TLS) para proteger os dados, evitando que informações sensíveis (como senhas) sejam interceptadas por terceiros.
