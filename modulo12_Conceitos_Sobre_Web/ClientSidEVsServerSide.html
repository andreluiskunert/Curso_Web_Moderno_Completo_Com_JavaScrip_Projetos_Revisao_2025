Resumo: Evolução das Arquiteturas de Aplicações

1. Mainframe

  • Tudo era processado no servidor central.
  • Terminais burros apenas exibiam e enviavam dados.
  • Fácil de atualizar (tudo concentrado).

2. Cliente-Servidor (Desktop)

  • Aplicações ricas feitas em Delphi, VB etc.
  • 2 camadas: cliente acessava diretamente o banco de dados.
  • 3 camadas: cliente + middleware + banco.
  • Dificuldade: atualizar clientes em todas as máquinas.

3. Web Server-side

  • O servidor gera HTML dinâmico e envia ao cliente.
  • Ex.: PHP, Java (JSP), Python (Django).
  • Vantagem: atualização centralizada (sem instalar em cada cliente).
  • Desvantagem: recarregamento da página, maior latência.

4. Web Client-side (SPA)

  • A lógica e a interface são processadas no navegador.
  • O servidor fornece apenas dados (JSON via APIs).
  • Ex.: React, Angular, Vue.js.
  • Experiência parecida com desktop: drag & drop, interatividade, atualizações rápidas.

5. Aplicações Mistas

  • Parte server-side + parte client-side.
  • Ex.: aplicação em PHP com algumas telas feitas em React.

Resumo Final

  • Server-side: servidor gera HTML e envia pronto ao cliente.
  • Client-side: servidor envia dados e o navegador monta a interface.
  • Evolução: Mainframe → Cliente-servidor → Server-side Web → Client-side Web (SPA).
